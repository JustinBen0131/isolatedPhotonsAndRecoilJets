# Inputs And Environment

## ROOT Environment

For any ROOT macro, `root`, `root-config`, ACLiC build, or ROOT-dependent
script in this repo, use the analysis environment wrapper:

```bash
./scripts/root_in_analysis_env.sh /Users/patsfan753/Desktop/analysis/env/bin/root -l -q 'macros/MyMacro.C()'
```

The wrapper reproduces `use_root_analysis`, forces the known-good RooUnfold at
`external/RooUnfold`, and avoids relying on interactive shell startup files.

## InputFiles

`InputFiles/` is a first-class project directory like `macros/`, `scripts/`,
`src/`, and `src_AuAu/`.

When Justin refers to local ROOT inputs, first check:

- `InputFiles/simEmbedded/`
- `InputFiles/simPhotonJet/`
- data-specific subdirectories under `InputFiles/`

The UI may not autocomplete `@InputFiles`; still use literal paths.

## Variant Inspector

When asked what input variants, cuts, or local merged outputs are available for
a dataset generated by `scripts/mergeRecoilJets.sh`, run:

```bash
./checkAvailableVariant <dataset>
```

Examples: `isSimEmbedded`, `isSimEmbeddedInclusive`, `isSim`, `isAuAu`,
`isPP`, `isPPrun25`.

The inspector is local-only. It parses local filenames and does not use ROOT,
SSH, or SFTP.

## Canonical Local SIM Products

Canonical local SIM combinations are built by the SFTP getter after downloads,
or on demand:

```bash
./scripts/sftp_get_recoiljets_outputs.sh mergeLocalSim all
```

Unless Justin explicitly asks to analyze individual SIM slices, use canonical
merged products:

- pp photon+jet 5+10+20:
  `dataOutput/combinedSimOnly/<cfg>/photonJet5and10and20merged_SIM/`
- embedded Au+Au photon+jet 12+20:
  `dataOutput/combinedSimOnlyEMBEDDED/<cfg>/photonJet12and20merged_SIM/`

Inclusive embedded jet 10+20 and inclusive jet5 canonical products are also
materialized under corresponding `combinedSimOnly*` folders when available.
